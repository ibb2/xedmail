Token behaviour — two states

While suggesting: plain colored text, no background. After Tab: locked into one cohesive highlighted token.

While suggesting (typing "from: sarah")

⌕ from: sarah@acme .co unread budget

Colored text, no backgrounds yet · Tab to lock in the suggestion

↓ Tab

After Tab — token locked in as one unit

⌕ from: sarah@acme.co unread budget

"from: sarah@acme.co" is now one solid highlighted token · "unread" locks in as you type it fully

Token locking rules

from: Locks in when Tab is pressed to accept the ghost contact, or when the user types a space after a full email address
unread/read Locks in immediately — exact keyword match, no ghost needed
today/date Locks in immediately — exact keyword match
